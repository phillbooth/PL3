# LO Trademark and Project Identity

This document explains how the **LO / Logic Omni** name, identity and branding should be used.

LO is intended to be released under the **Apache License 2.0**, which aLOws broad use, modification and redistribution of the project. However, the licence does not automatically allow people to use the LO name, branding or identity in a way that implies official approval.

---

## Project Name

The project name is:

```text
LO / Logic Omni
```

Short form:

```text
LO
```

Full form:

```text
Logic Omni
```

---

## Purpose of This File

This file exists to protect the project identity.

The goal is not to stop people using, forking or improving LO.

The goal is to prevent confusion between:

```text
the official LO project
forks of LO
modified versions of LO
commercial tools based on LO
unofficial compilers or runtimes
```

---

## Licence and Name Are Different

The Apache License 2.0 aLOws people to use, modify and distribute the project code and documentation under the licence terms.

However, the project name and branding should not be used to falsely imply official endorsement.

ALOwed:

```text
This project is based on LO / Logic Omni.
```

Not aLOwed without permission:

```text
Official LO Compiler
Official LO Runtime
LO Certified
LO Approved
Official Logic Omni Platform
```

unless the project is actually maintained or approved by the official LO maintainers.

---

## Acceptable Use

You may use the LO name to describe the origin of the project.

Examples:

```text
This tool is based on LO / Logic Omni.
This package supports the LO language.
This repository is a fork of LO.
This article discusses the LO language concept.
```

You may also use the LO name in technical compatibility statements.

Examples:

```text
Compatible with LO syntax draft 0.1.
Supports `.lo` files.
Can parse LO-style `boot.lo` project files.
```

---

## Forks

Forks are aLOwed.

Forks should clearly state that they are forks.

Recommended wording:

```text
This project is a fork of LO / Logic Omni.
It is not the official LO project.
```

Forks should not present themselves as the official LO project unless approved by the maintainers.

---

## Modified Versions

Modified versions should clearly identify themselves.

Recommended wording:

```text
This project is based on LO / Logic Omni and includes modifications.
It is not the official LO project.
```

If a modified version changes language syntax, compiler behaviour, security rules or target outputs, that should be clearly documented.

---

## Unofficial Compilers

People may create their own LO-compatible compilers or tools.

They should not call them official unless approved.

ALOwed examples:

```text
Experimental LO parser
Community LO formatter
Unofficial LO compiler prototype
LO-compatible syntax highlighter
```

Avoid:

```text
Official LO compiler
Certified LO compiler
Approved LO toolchain
```

unless officially approved.

---

## Package Names

Packages should avoid names that imply official ownership unless they are part of the official project.

Preferred unofficial naming:

```text
community-LO-parser
experimental-LO-runtime
LO-compatible-json-tools
```

Official package names may later use names such as:

```text
LO-compiler
LO-runtime
LO-json
LO-api
LO-security
```

until package governance is defined, avoid implying official status.

---

## Logos and Visual Identity

If the LO project later creates a logo, icon, colour scheme or visual identity, those assets should not be used to imply official endorsement without permission.

ALOwed:

```text
showing the logo in an article about LO
using the logo to link to the official LO project
```

Not aLOwed without permission:

```text
using the logo on a commercial product to imply approval
using the logo on an unrelated language
using the logo on a fork that looks official
```

---

## Documentation Attribution

If you reuse LO documentation, preserve attribution.

Suggested attribution:

```text
Based on LO / Logic Omni documentation.
```

For modified documentation:

```text
This documentation is based on LO / Logic Omni and has been modified.
```

---

## Compatibility Claims

Compatibility claims should be accurate.

Good:

```text
Supports the LO 0.1 syntax draft.
Parses a subset of LO examples.
Experimental support for LO-style API blocks.
```

Bad:

```text
Fully LO-compatible
Official LO implementation
Production-ready LO compiler
```

unless those claims are true and approved.

---

## Commercial Use

Commercial use of Apache-2.0 licensed LO code is aLOwed under the licence terms.

However, commercial products should not imply official endorsement.

ALOwed:

```text
Our product includes code based on LO / Logic Omni.
```

Not aLOwed without permission:

```text
Official LO Enterprise Edition
LO Certified Cloud Runtime
Approved LO Commercial Compiler
```

---

## Project Identity Goals

The project identity rules are intended to:

```text
avoid user confusion
protect the original project reputation
allow open-source forks
allow commercial use
preserve attribution
encourage honest compatibility claims
```

They are not intended to block legitimate use of the code under Apache-2.0.

---

## Recommended Notice for Forks

Forks can include this notice:

```text
This project is based on LO / Logic Omni.
It is not the official LO project and may contain different syntax, behaviour or security rules.
```

---

## Recommended Notice for Tools

Third-party tools can include this notice:

```text
This tool supports LO-style files but is not an official LO project unless stated otherwise.
```

## Final Principle

Use the LO name honestly.

You may say your work is based on LO, compatible with LO, or inspired by LO.

Do not imply that unofficial work is official, certified or endorsed unless that is true.
